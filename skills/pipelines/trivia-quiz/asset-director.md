# Asset Director — Trivia Quiz Pipeline

## When To Use

After `script` produces `script.json`. You are resolving 3 backdrop stills
to local files and writing `asset_manifest.json`.

## v0.1: FLUX-Generated Stills

For v0.1, backdrops are generated by the `flux_image` tool — one image per
question, themed to the question topic. Costs ~$0.03 per image ($0.09 per
post). OpenArt motion clips remain a v0.3 target if image+KenBurns doesn't
hold viewer attention enough.

## Process

### 1. Derive Backdrop Prompts from `brief.json`

For each question (Q1/Q2/Q3), build a FLUX prompt from the question text and
surprise fact. The prompt should produce a topical PHOTOREAL still — never
people-portraits (that's ellie's lane), never on-screen text (Remotion draws
the UI). Example template:

```
photorealistic {scene}, cinematic, golden hour lighting, no people no text,
shot on Sony A7, depth of field, 9:16 vertical, magazine-quality
```

Where `{scene}` is a 2–5 word summary of the question topic — pulled from
the brief's `topic_mix` or hand-authored per row in `quiz_row.yaml::qN.backdrop_hint`.

### 2. Call `flux_image` Three Times

```python
from tools.tool_registry import registry
registry.discover()
flux = registry._tools["flux_image"]

for qid, prompt in [("q1", q1_prompt), ("q2", q2_prompt), ("q3", q3_prompt)]:
    result = flux.execute({
        "prompt": prompt,
        "width": 1024,
        "height": 1820,                  # 9:16 portrait
        "model": "flux-pro/v1.1",
        "output_path": f"projects/trivia-quiz/{slug}/assets/images/{qid}_bg.jpg",
    })
    assert result.success, f"FLUX failed on {qid}: {result.error}"
```

**Read the Layer 3 skill first** — `.agents/skills/flux-best-practices/` and
`bfl-api/` per `flux_image.agent_skills`. Skipping that step produces generic
prompts that miss model-specific quality wins.

### 3. Validate Each Still

Use `ffprobe` to confirm each generated image is 1024×1820 (or whatever
dimensions FLUX returned). Reject anything that came back smaller — Ken
Burns zoom-in on a low-res still looks washed out.

### 2. Validate Each Still

Use `ffprobe` to confirm each image is at least 1080×1920 (portrait) OR
1920×1080+ (landscape — `assemble_quiz.py` will crop to portrait). Reject
anything smaller — Ken Burns zoom-in on a low-res still looks washed out.

### 3. Write `asset_manifest.json`

```json
{
  "metadata": {
    "assets": {
      "q1_bg": {
        "kind": "still",
        "path": "projects/trivia-quiz/<slug>/assets/images/q1_bg.jpg",
        "dimensions": [w, h],
        "kenburns": { "zoom_from": 1.0, "zoom_to": 1.08, "pan": "left-to-right" }
      },
      "q2_bg": { ... },
      "q3_bg": { ... }
    }
  }
}
```

Ken Burns defaults: `zoom_from=1.0`, `zoom_to=1.08`, alternating pan direction
per question to keep the motion varied (Q1 → left-to-right, Q2 → top-to-bottom,
Q3 → right-to-left). Override per-asset only if the still's composition
demands it (e.g., subject is near a frame edge).

## Output

`projects/trivia-quiz/<slug>/artifacts/asset_manifest.json` — 3 image assets keyed
`q1_bg`, `q2_bg`, `q3_bg`.

## Checkpoint

`human_approval_default: false`. Auto-proceed if all 3 stills are present
and valid. Stop only if a still is missing or fails ffprobe.
